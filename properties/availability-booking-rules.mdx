Cendra syncs availability settings and booking rules from your property management system, making this data available to AI agents for accurate guest responses about availability inquiries.

## Availability Settings

| Setting | Description |
|---------|-------------|
| **Default availability** | Whether the property is available for booking by default |
| **Booking window** | How far in advance guests can book |
| **Advance notice** | Minimum notice before a booking can be made |
| **Preparation time** | Buffer between reservations for cleaning and turnover |
| **Check-in restrictions** | Specific days or times when check-in is allowed |
| **Maximum nights per year** | Annual cap on rental nights (for regulatory compliance) |

## Length of Stay Limits

| Setting | Description |
|---------|-------------|
| **Minimum stay** | Shortest allowed reservation (e.g., 2 nights minimum) |
| **Maximum stay** | Longest allowed reservation (e.g., 30 nights maximum) |

## How AI Uses Availability Data

When a guest asks "is the property available next weekend?", Cendra's AI:
1. Checks the property's calendar availability from the PMS
2. Verifies the request meets booking rules (minimum stay, advance notice)
3. Responds with accurate availability information

This eliminates manual calendar checks and ensures guests get instant, accurate answers.
